Designing Resilient Microservices: A Deep Dive into Event-Sourcing

Published Oct 12, 2023 • 15 min read • By Nguyen Van Hai

The shift towards microservices has introduced unprecedented scalability, but it also brought a new class of distributed systems challenges. Chief among these is maintaining data consistency across independent services.

The Fallacy of Distributed Transactions

In monolithic systems, ACID transactions were our safety net. In a distributed world, the two-phase commit (2PC) is often too slow and brittle. This is where Event Sourcing changes the paradigm: instead of storing the current state, we store the sequence of events that led to it.

EventStore.cs 
public async Task AppendEvent(Guid streamId, Event @event) 
{
    var sequence = await _db.GetLastSequence(streamId);
    var record = new EventRecord(
        streamId, 
        sequence + 1, 
        @event.GetType().Name, 
        JsonConvert.SerializeObject(@event)
    );
    await _db.Events.AddAsync(record);
    await _db.SaveChangesAsync();
}

Eventual Consistency in Practice

Transitioning to eventual consistency requires a mental shift for both developers and stakeholders. We must design for the "in-between" states. When a user places an order, it isn't "Confirmed" immediately; it is "PendingValidation".

Key Architectural Pattern

"The Single Source of Truth is no longer a database table; it is the Append-Only Log."

Using a message broker like Apache Kafka or RabbitMQ, we can ensure that every downstream service reacts to these events at its own pace, providing high availability even if some parts of the system are temporarily offline.

Architect's Note

Event Sourcing isn't a silver bullet. It introduces complexity in querying (CQRS becomes mandatory) and data evolution (versioning events). Only use it for high-value core domains.

Technical Glossary

What is Kafka?

A distributed event store and stream-processing platform that can handle trillions of events a day.

CQRS

Command Query Responsibility Segregation. Separating the model for updating data from the model for reading data.
