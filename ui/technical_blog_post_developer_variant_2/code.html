Scaling LLM Inferencing at the Edge

When architecting modern microservices that leverage local LLM inferencing, the challenge isn't just about the model—it's about the orchestration layer. In this deep dive, we explore how to bridge Golang's concurrency model with Ollama's local hosting capabilities on Azure AKS.

01. Resource Allocation & Scheduling

The core bottleneck was memory fragmentation during high-concurrency inference requests. We implemented a custom pool of Ollama instances, each bound to specific node affinity in Azure.

inference_pool.go 
type OllamaPool struct {
    instances []*Instance
    mu        sync.RWMutex
    strategy  LoadBalancerStrategy
}

// Allocate finds the next available GPU-optimized node
func (p *OllamaPool) Allocate(ctx context.Context) (*Instance, error) {
    p.mu.Lock()
    defer p.mu.Unlock()
    
    for _, inst := range p.instances {
        if inst.Status == StatusReady {
            inst.Status = StatusBusy
            return inst, nil
        }
    }
    return nil, fmt.Errorf("no available instances")
}
